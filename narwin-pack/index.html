narwin-pack

And why we built a PostCSS package

Cory Tanner / @ctannerweb / @dockyard

Why PostCSS?

  • The main reason to use PostCSS is the ability to pick and choose plugins you want to use
  • You can set PostCSS up to act like Sass
  • Is modular so the preprocessor is 3-30 times faster
  • Create custom Plugin Packs that you can easily use in multiple projects like we did with narwin-pack

PostCSS Plugins

  • Take normal CSS and modifies it based on the rules in the plugin
  • You can write plugins yourself
  • Don't expect one plugin to solve every need
  • The beauty with plugins is that the community is always making new plugins that solve new problems

PostCSS Packs

  • A collection of plugins that make up a prepackaged development process
  • cssnext allows you to use tomorrow's CSS syntax now
  • narwin-pack is a package with hand picked plugins that meet the needs of DockYard's UX development process

narwin-pack Plugins

  • autoprefixer
  • postcss-partial-import
  • postcss-nested
  • postcss-custom-properties
  • postcss-calc

autoprefixer

  • Based on caniuse rules this plugin will auto add the needed prefix(es) of a style
  • You can configure browser support
  • Very helpful with transforms and transitions
  • .block { display: flex; transition: all .5s; } .block { display: -webkit-box; display: -webkit-flex; display: -ms-flexbox; display: flex; -webkit-transition: all .5s; transition: all .5s; }

postcss-partial-import

  • Lets you set up modular CSS files and import them all into one app.css file
  • When you combine modular CSS files into one file you only have one http request for CSS files
  • Who doesn't like modular CSS!
  • app.css @import "type"; @import "load"; @import "modules/header"; @import "modules/footer";

postcss-nested

  • Unwraps nested rules how Sass would
  • We only use this for pseudo elements/classes and media queries
  • You lose search options if you nest BEM classes like .block { background-color: var(--green); &__element { color: var(--green); } } .block { background-color: var(--green); } .block__element { color: var(--green); }

postcss-custom-properties

  • Variables in your CSS!
  • We have a load.css file that has all our variables :root { --green: rgb(61, 154, 104); } color: var(--green); color: rgb(61, 154, 104);

postcss-calc

  • Reduces calc() references when possible
  • Any calculations with variables are now possible :root { --w-l-wrap: 980px; } width: calc(var(--w-l-wrap) / 2);
